Font Loading Test for Najwa's Website

Solution to Your Font Issue

Based on your site structure, you should use one of these paths:

/* Option 1: Relative path (if CSS is in same folder as fonts folder) */
src: url('fonts/ebrima.ttf') format('truetype');

/* Option 2: Absolute path from root */
src: url('/Najwa/fonts/ebrima.ttf') format('truetype');

Try changing your font path to: url('fonts/ebrima.ttf')

Testing Ebrima Font

If the font is loaded correctly, the text below will appear in Ebrima:

Normal Weight (should use ebrima.ttf):
The quick brown fox jumps over the lazy dog.
نجوا نظام - مصممة الديكور الداخلي
Bold Weight (should use ebrima-bold.ttf):
The quick brown fox jumps over the lazy dog.
نجوا نظام - مصممة الديكور الداخلي

Note: If you see the same font in both boxes, the bold version may not be loading correctly, or your browser may be synthesizing the bold style.

Correct @font-face Implementation

Here's how your CSS should look:

@font-face {
  font-family: 'Ebrima';
  src: url('fonts/ebrima.ttf') format('truetype');
  font-weight: normal;
  font-style: normal;
}

@font-face {
  font-family: 'Ebrima';
  src: url('fonts/ebrima-bold.ttf') format('truetype');
  font-weight: bold;
  font-style: normal;
}

body {
  font-family: 'Ebrima', sans-serif;
}
